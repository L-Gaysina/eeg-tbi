# EEG-TBI Preprocessing Pipeline

Пайплайн предобработки и базового контроля качества ЭЭГ-данных для исследования последствий черепно-мозговой травмы (ЧМТ).

Репозиторий содержит код, описание методики и безопасные агрегированные примеры результатов. Сырые ЭЭГ-сигналы, полные файлы эпох, клинические метаданные и приватные пути к данным **не публикуются**.

## О проекте

Цель данного этапа — привести разнородные ЭЭГ-записи пациентов с ЧМТ и контрольной группы к единому техническому формату для последующего количественного анализа и машинного обучения.

Пайплайн решает задачу **подготовки данных**, а не полного анализа ЭЭГ. Он формирует стандартизированные записи, унифицирует каналы, рассчитывает базовые метрики качества и сохраняет таблицу записей, пригодных для дальнейшей обработки.

Основной результат этапа:

```text
analysis_ready_inventory.csv
```

Этот файл используется как вход для следующего этапа — расчёта спектральных, временных, событийных и сетевых признаков ЭЭГ.

---

## Что делает пайплайн

Основной ноутбук:

```text
notebooks/01_prepare_eeg_dataset.ipynb
```

Он выполняет следующие шаги:

1. Проверяет локальные пути к исходным данным.
2. Формирует inventory-таблицы для группы ЧМТ и контрольной группы.
3. Выбирает контрольные записи OpenNeuro `ds005385` по заданному протоколу.
4. Загружает клинические `.mat`-файлы и контрольные EDF-файлы.
5. Преобразует записи в формат `MNE`.
6. Выполняет ресемплинг до 500 Гц.
7. Применяет полосовую фильтрацию и notch-фильтр.
8. Выполняет common average reference.
9. Проверяет или формирует 4-секундные эпохи.
10. Стандартизирует названия каналов.
11. Определяет общий набор каналов между группами.
12. Приводит все записи к единому набору каналов.
13. Рассчитывает базовые QC-метрики.
14. Исключает технически ненадёжные записи.
15. Сохраняет итоговые таблицы и демонстрационные графики контроля качества.

Общая схема:

```text
локальные исходные файлы
        ↓
inventory
        ↓
стандартизация MNE Epochs
        ↓
унификация каналов
        ↓
базовый контроль качества
        ↓
analysis_ready_inventory.csv
```

## Данные

Пайплайн использует два источника данных.

### 1. Группа ЧМТ

Клинические ЭЭГ-записи пациентов с черепно-мозговой травмой представлены локальными файлами MATLAB/EEGLAB:

```text
/path/to/local/tbi_mat_files/
    *.mat
```

Ожидается, что внутри `.mat`-файлов содержится структура `EEG`, включающая, при наличии:

- `EEG.data` — массив ЭЭГ-сигнала;
- `EEG.srate` — частоту дискретизации;
- `EEG.chanlocs` — информацию о каналах;
- `EEG.bad_chans` — предварительно размеченные плохие каналы;
- `EEG.bad_epochs` — предварительно размеченные плохие эпохи.

Клинические данные являются закрытыми исследовательскими данными. Они были обезличены и использовались только в рамках исследования, но не могут быть опубликованы из-за институциональных ограничений, требований конфиденциальности и условий доступа.

### 2. Контрольная группа

Контрольные записи берутся из открытого датасета OpenNeuro `ds005385` в формате EDF:

```text
/path/to/local/openneuro/ds005385/
    sub-*/ses-*/eeg/*.edf
```

В репозитории EDF-файлы не дублируются, так как они занимают много места и доступны из внешнего источника. Для запуска пайплайна пользователь должен самостоятельно скачать датасет и указать локальный путь к нему.

Для основного анализа используется протокол:

```text
EyesClosed / acq-pre
```

Он выбран как наиболее воспроизводимый вариант resting-state EEG и наиболее близкий к типичной клинической фоновой записи.

## Настройка конфигурации

Пути и основные параметры задаются в словаре `CONFIG` внутри ноутбука.

Пример:

```python
from pathlib import Path

CONFIG = {
    "tbi_base_dir": Path("/path/to/local/tbi_mat_files"),
    "ctl_ds005385_root": Path("/path/to/local/openneuro/ds005385"),
    "work_dir": Path("/path/to/work_dir"),

    "tbi_mat_glob": "**/*.mat",

    "target_sfreq": 500.0,
    "l_freq": 0.5,
    "h_freq": 70.0,
    "notch_freqs": [50, 100, 150, 200],
    "ref": "average",

    "save_standardized_epochs": True,

    "min_epochs_per_record": 10,
    "drop_epoch_p2p_uv": 350.0,
    "max_bad_epoch_prop": 0.5,
}
```

В публичной версии репозитория реальные локальные пути заменены на шаблонные.

## Запуск

1. Клонировать репозиторий:

```bash
git clone https://github.com/YOUR_USERNAME/eeg-tbi-preprocessing.git
cd eeg-tbi-preprocessing
```

2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Скачать контрольный датасет OpenNeuro `ds005385` и сохранить его локально.

4. Открыть основной ноутбук:

```text
notebooks/01_prepare_eeg_dataset.ipynb
```

5. Указать локальные пути к данным в `CONFIG`.

6. Запустить ноутбук сверху вниз.

## Параметры предобработки

| Параметр | Значение |
|---|---:|
| Частота дискретизации после стандартизации | 500 Гц |
| Полосовой фильтр | 0.5–70 Гц |
| Notch-фильтр | 50 Гц и гармоники |
| Референс | Common average reference |
| Длительность эпохи | 4 секунды |
| Минимальное число эпох на запись | 10 |
| Peak-to-peak порог | 350 мкВ |
| Максимальная допустимая доля плохих эпох | 0.5 |

## Унификация каналов

После загрузки и стандартизации данных определяется общий набор ЭЭГ-каналов, присутствующий в обеих группах.

В демонстрационном запуске:

```text
ЧМТ до унификации: 60 каналов
Контроль до унификации: 64 канала
Общий набор после пересечения: 58 каналов
```

Все дальнейшие этапы анализа используют только единый набор каналов. Это необходимо для корректного расчёта признаков и сопоставимости групп.

Список общих каналов сохраняется в файле:

```text
outputs_example/tables/common_channels.txt
```

---

## Контроль качества

После унификации каналов выполняется базовый контроль качества. Проверяются:

- число успешно обработанных записей;
- число эпох на запись;
- частота дискретизации;
- длительность эпох;
- число каналов;
- peak-to-peak амплитуда;
- доля эпох, превышающих p2p-порог.

Запись исключается из дальнейшего анализа, если выполняется хотя бы одно из условий:

1. число эпох меньше `min_epochs_per_record`;
2. доля эпох, превышающих p2p-порог, больше `max_bad_epoch_prop`.

В демонстрационном запуске использовались критерии:

```text
min_epochs_per_record = 10
drop_epoch_p2p_uv = 350.0
max_bad_epoch_prop = 0.5
```

Пример итоговой сводки после QC:

| Группа | Субъекты | Записи | Эпохи | Каналы |
|---|---:|---:|---:|---:|
| ЧМТ | 89 | 199 | 22 863 | 58 |
| Контроль | 72 | 356 | 16 470 | 58 |

## Ограничения

Данный пайплайн решает задачу технической подготовки данных и базового контроля качества. Он не устраняет все возможные методические ограничения исследования.

Ключевые ограничения:

- данные ЧМТ и контрольной группы получены из разных источников;
- различия в длительности записей могут влиять на стабильность последующих признаков;
- амплитудный QC является базовым фильтром и не заменяет экспертную оценку качества ЭЭГ;
- унификация каналов повышает сопоставимость групп, но может приводить к исключению части доступной пространственной информации;
- дальнейшая интерпретация результатов должна учитывать возможные различия в протоколах регистрации и источниках данных.

## Назначение репозитория

Репозиторий сопровождает исследовательский проект по анализу ЭЭГ в состоянии покоя у пациентов с ЧМТ. Код предоставлен для демонстрации вычислительного пайплайна, повышения прозрачности методики и облегчения воспроизведения этапа подготовки данных на локально доступных источниках.

Сырые клинические ЭЭГ-записи не являются публично доступными.

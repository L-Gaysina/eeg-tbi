# EEG-TBI

Анализ количественных признаков ЭЭГ у пациентов с черепно-мозговой травмой и здоровых испытуемых с использованием методов обработки сигналов, статистического анализа и машинного обучения.

## О проекте

Репозиторий содержит код и репрезентативные результаты анализа resting-state ЭЭГ у пациентов с черепно-мозговой травмой (ЧМТ) и контрольной группы.

Цель проекта — выделить многоуровневые количественные признаки ЭЭГ, отражающие посттравматические изменения нейрофизиологической организации сигнала, и оценить их информативность для межгрупповой классификации.

Анализ включает:

* предобработку ЭЭГ и контроль качества;
* расчёт спектральных признаков;
* анализ временной структуры амплитудной огибающей;
* оценку функциональной связности и графовых метрик;
* анализ транзиентных событий ЭЭГ;
* статистическое сравнение групп;
* построение моделей машинного обучения.

### 3. Машинное обучение

### Раздел [`03_machine_learning/`](./03_machine_learning/)   
 предназначен для оценки информативности рассчитанных признаков и построения моделей межгрупповой классификации.

 Cодержание:

* формирование итоговой feature matrix;
* subject-level train/test split;
* сравнение блоков признаков;
* оценка моделей с учётом дисбаланса классов;
* ROC-AUC, balanced accuracy, sensitivity, specificity;
* permutation-тесты;
* анализ компактных моделей;
* интерпретация наиболее информативных признаков.

### Статистический подход

Основная статистическая интерпретация проводится на уровне субъектов.

Если у одного субъекта было несколько записей, признаки предварительно агрегировались внутри субъекта, чтобы избежать псевдорепликации.

Для межгрупповых сравнений использовались:

* критерий Манна–Уитни;
* размер эффекта Hedges’ g;
* коррекция на множественные сравнения методом Benjamini–Hochberg FDR.

Положительные значения Hedges’ g соответствуют большим значениям признака в группе ЧМТ, отрицательные — большим значениям в контрольной группе.

## Данные

Сырые клинические ЭЭГ-записи пациентов с ЧМТ не включены в репозиторий из-за ограничений доступа и требований конфиденциальности.

Контрольные данные должны быть получены из исходного публичного набора данных.

В репозитории публикуются только:

* код анализа;
* репрезентативные рисунки;
* агрегированные статистические таблицы;
* анонимизированные примеры таблиц признаков;
* примеры выходов контроля качества.

Полные таблицы признаков с внутренними идентификаторами субъектов и записей не публикуются.
## Термины и обозначения
| Обозначение       | Расшифровка                                                                           |
| ----------------- | ------------------------------------------------------------------------------------- |
| EEG / ЭЭГ         | Электроэнцефалография                                                                 |
| TBI / ЧМТ         | Черепно-мозговая травма                                                               |
| PSD               | Спектральная плотность мощности                                                       |
| SEF50 / SEF95     | Спектральная граничная частота, ниже которой сосредоточено 50% / 95% мощности спектра |
| FDR               | Коррекция на долю ложных открытий                                                     |
| ROI               | Область интереса                                                                      |
| DFA               | Анализ детрендированных флуктуаций                                                    |
| wPLI              | Взвешенный индекс фазовой задержки                                                    |
| PLV               | Индекс фазовой синхронизации                                                          |
| ROC-AUC           | Площадь под ROC-кривой                                                                |
| Balanced accuracy | Сбалансированная точность                                                             |

## Как воспроизвести анализ
1. Выполнить ноутбук предобработки в [`01_preprocessing/`](./01_preprocessing/) 

2. Убедиться, что сформирован очищенный inventory:
```text
analysis_ready_inventory_cleaned.csv
```
3. Выполнить ноутбуки расчёта количественных признаков в [`02_quantitative_eeg_features/`](./02_quantitative_eeg_features/) 

4. После формирования итоговой таблицы признаков выполнить ML-анализ в [`03_machine_learning/`](./03_machine_learning/)   

Перед запуском ноутбуков необходимо указать локальные пути к данным в конфигурационных ячейках.
